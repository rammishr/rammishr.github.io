Architectural Patterns

This covers a wide range of architectural patterns present in software industry.

Circuit Breaker
Circuit Breaker

In microservice ecosystem services are mostly dependent on other remote services. These ecosystem should take into consideration that these remote services could fail. They could fail because of server faults, application load, network issues, etc. In such scenario, it is pointless to keep retying. Circuit breaker pattern can be useful in such scenarios. This pattern instead of retying trips the calls and returns a fallback response till remote service is restired.

Sidecar
Sidecar

The Sidecar Pattern is a software architecture pattern in which an application is decomposed into two separate parts: a primary component and a secondary. The primary component is responsible for performing the application's core functionality, while the sidecar provides additional features, such as monitoring, logging, or data processing. This pattern is useful in distributed systems where a single application needs to perform multiple tasks, but each task requires different resources and dependencies.
